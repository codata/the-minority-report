**Detailed Summary of the File**

The file `semantic_croissant.json` is a **central Semantic Croissant metadata catalog** (conforming to Croissant 1.1) for the **UNDRR-ISC Hazard Information Profiles (HIPs) multilingual translation project**, created under the CODATA "Minority Report" initiative.

### Core Purpose and Structure
- **@type**: `sc:DataCatalog` (Schema.org Data Catalog with Croissant extensions).
- **Name**: "UNDRR-ISC Hazard Information Profiles — Multilingual Translation Semantic Croissant Catalog".
- **Scope**: It serves as an index and navigation hub for ~282 individual HIPS datasets (covering hazards like airborne diseases, earthquakes, floods, cyberattacks, etc.). It references per-hazard Croissant files, translations, original sources, and derived artifacts.

Key elements include:
- **@context**: Defines namespaces (Schema.org, Croissant `cr:`, DC Terms, ODRL for rights, CDIF, etc.) and custom processing like `bs4ExtractionPattern` for HTML parsing via BeautifulSoup.
- **Description** (extensive): Details the catalog's content — translated terminology CSVs, original HTML pages (embedded or linked, with clean plaintext extracts), SKOS/JSON-LD provenance, and metadata. It was generated by a multi-LLM pipeline (Gemini, GPT, DeepSeek, Gemma) with consensus arbitration. Includes BS4 extraction patterns for titles, dates, and article content from PreventionWeb HTML pages.
- **License**: CC BY 4.0.
- **Version/Date**: 1.0.0, published 2026-06-14.
- **Publisher**: CODATA / The Minority Report Project.
- **Keywords**: UNDRR, HIPS, hazard information profiles, multilingual, controlled vocabulary, disaster risk, translations, NLP, AI dataset, Croissant.
- **cr:recordSet**: Indexes all datasets (total ~282, with ~275 having embedded HTML).
- **potentialAction**: Defines a SearchAction for lookups by HIPS code (e.g., BI0101) and language code (custom codes like `ua` for Ukrainian, `ch` for Chinese, `dk` for Danish).
- **dataset array**: The bulk of the file. Each entry is a `sc:Dataset` for one HIPS concept (e.g., BI0101_Airborne_Diseases, BI0102_Bloodborne_Viruses). These include:
  - Descriptions of translations (term, context, translation, language, confidence, model, consensus, version).
  - `isBasedOn` references: Original HTML, clean TXT extracts, and sometimes semantic metrics JSON.
  - SHA256 hashes for integrity.
  - `did` (decentralized identifier) with provenance, available languages, and instructions for models.
  - Links to per-hazard `semantic_croissant.json` files and raw GitHub resources.

**Additional Features**:
- **Provenance & Integrity**: Emphasizes UNF-6 fingerprints, SHA256, and instructions for models to cite authoritative sources.
- **ODRL Policy**: Links to rights/policy JSON-LD.
- **Transparency**: Notes LLM-generated translations; directs to README-AI-CONSUMPTION.md for usage.
- **Extraction Guidance**: Detailed BeautifulSoup patterns for parsing original PreventionWeb pages (e.g., targeting `<article class='custom-full-content'>`).

The catalog enables structured, machine-readable access to standardized hazard definitions, multilingual translations, and provenance for disaster risk reduction (aligned with Sendai Framework, SDGs, etc.). HIPs cover categories like biological, meteorological/hydrological, geological, chemical, etc. (roughly 281 hazards in the 2025 update).

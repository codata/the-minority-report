# Training LLMs with Minority Report Data

This guide explains how to use the high-quality, consensus-validated multilingual datasets generated by **Minority Report** to train or fine-tune Large Language Models (LLMs).

## Data Overview

The generated **Croissant** files (JSON-LD) contain precise, technical term-concept pairs enriched with:
1.  **Consensus Translations**: Terms validated by multiple models (e.g., `gpt-oss`, `gemma3`).
2.  **Model Provenance**: Metadata indicating which model generated each translation.
3.  **Semantic Context**: Definitions/Scope notes for each term.

### Structure Example
```json
{
  "@context": { ... },
  "@type": "sc:Dataset",
  "name": "Peste des petits ruminants",
  "description": "A highly contagious disease of goats and sheep...",
  "sc:alternateName": [
    {
      "@value": "Peste des petits ruminants",
      "@language": "fr",
      "creator": { "@id": "model_gpt-oss_latest" }
    },
    {
      "@value": "PPR",
      "@language": "ru",
      "creator": { "@id": "model_deepseek-r1_14b" }
    }
  ]
}
```

## Use Case 1: Supervised Fine-Tuning (SFT)

The primary use case is `Context` -> `Multilingual Term` translation.

### Data Extraction Strategy
Target format for SFT:
```json
{"messages": [
  {"role": "system", "content": "You are a specialized translator for disaster risk terminology."},
  {"role": "user", "content": "Translate 'Peste des petits ruminants' to Russian. Context: A highly contagious disease..."},
  {"role": "assistant", "content": "ППР"}
]}
```

### Loading Data (Python)
You can directly parse the JSON-LD files to create a dataset:

```python
import json
import glob

dataset = []
for file_path in glob.glob("output/croissant_*.json"):
    with open(file_path, "r") as f:
        data = json.load(f)
        
        term = data["name"]
        context = data["description"]
        
        # Extract translations
        for item in data.get("sc:alternateName", []):
            lang = item["@language"]
            translation = item["@value"]
            model_id = item.get("creator", {}).get("@id", "unknown")
            
            # Filter for specific languages if needed
            dataset.append({
                "instruction": f"Translate '{term}' to {lang} based on this definition: {context}",
                "input": context,
                "output": translation,
                "provenance": model_id
            })

print(f"Loaded {len(dataset)} training examples.")
```

## Use Case 2: Preference Optimization (DPO/ORPO)

Since our pipeline uses **Consensus** and **Arbitration**, the final result in the Croissant file represents the "Chosen" response.

To build a DPO dataset, you would need to log the *rejected* candidates during the orchestration phase (currently, `orchestrator.py` logs rejection to stdout but stores only the winner). 

**Future Strategy for DPO:**
1.  Modify `orchestrator.py` to save `rejected_candidates` in the JSON output.
2.  Construct pairs:
    *   **Chosen**: The consensus translation.
    *   **Rejected**: The minority vote or the candidate lost in arbitration.

## Use Case 3: Model Distillation

You can use the provenance data to train smaller models to mimic larger ones.
*   **Teacher**: `gpt-oss:latest` (or the consensus result).
*   **Student**: `gemma3:4b` or `llama3:8b`.

Filter the dataset where `"provenance"` indicates a high-quality model or a consensus win, and train the smaller model on these high-confidence examples.

## Fine-Tuning Execution Guide

This section provides a concrete workflow for fine-tuning a base model (e.g., Llama 3 or Gemma 2) using the dataset extracted above. We recommend using **Unsloth** for efficiency (2-5x faster, 70% less memory).

### 2. Training Script (Example)
Save this as `train.py`:

```python
from unsloth import FastLanguageModel
import torch
from datasets import Dataset
import json
import glob

# 1. Load Data from Croissant (Minority Report)
training_data = []
for file_path in glob.glob("output/croissant_*.json"):
    with open(file_path, "r") as f:
        data = json.load(f)
        term = data["name"]
        context = data["description"]
        # Extract Identifier (HIPS Code)
        identifier = data.get("https://schema.org/identifier", "")
        
        for item in data.get("sc:alternateName", []):
            lang = item["@language"]
            translation = item["@value"]
            # Format as Alpaca/Instruction style
            output_text = translation
            if identifier:
                output_text += f" (HIPS: {identifier})"
                
            training_data.append({
                "instruction": f"Translate '{term}' to {lang} based on the provided context. Include the HIPS code if available.",
                "input": context,
                "output": output_text
            })

dataset = Dataset.from_list(training_data)

# 2. Load Base Model
max_seq_length = 2048
model, tokenizer = FastLanguageModel.from_pretrained(
    model_name = "unsloth/gemma-2-9b-it", # or unsloth/llama-3-8b-Instruct
    max_seq_length = max_seq_length,
    dtype = None,
    load_in_4bit = True,
)

# 3. Add LoRA Adapters
model = FastLanguageModel.get_peft_model(
    model,
    r = 16,
    target_modules = ["q_proj", "k_proj", "v_proj", "o_proj", "gate_proj", "up_proj", "down_proj"],
    lora_alpha = 16,
    lora_dropout = 0,
    bias = "none",
    use_gradient_checkpointing = True,
)

# 4. Format Prompt
alpaca_prompt = """Below is an instruction that describes a task, paired with an input that provides further context. Write a response that appropriately completes the request.

### Instruction:
{}

### Input:
{}

### Response:
{}"""

def formatting_prompts_func(examples):
    instructions = examples["instruction"]
    inputs       = examples["input"]
    outputs      = examples["output"]
    texts = []
    for instruction, input, output in zip(instructions, inputs, outputs):
        text = alpaca_prompt.format(instruction, input, output) + tokenizer.eos_token
        texts.append(text)
    return { "text" : texts, }

dataset = dataset.map(formatting_prompts_func, batched = True)

# 5. Train
from trl import SFTTrainer
from transformers import TrainingArguments

trainer = SFTTrainer(
    model = model,
    tokenizer = tokenizer,
    train_dataset = dataset,
    dataset_text_field = "text",
    max_seq_length = max_seq_length,
    dataset_num_proc = 2,
    args = TrainingArguments(
        per_device_train_batch_size = 2,
        gradient_accumulation_steps = 4,
        warmup_steps = 5,
        max_steps = 60,
        learning_rate = 2e-4,
        fp16 = not torch.cuda.is_bf16_supported(),
        bf16 = torch.cuda.is_bf16_supported(),
        logging_steps = 1,
        optim = "adamw_8bit",
        weight_decay = 0.01,
        lr_scheduler_type = "linear",
        seed = 3407,
        output_dir = "outputs",
    ),
)

trainer.train()

# 6. Export to GGUF (for Ollama)
model.save_pretrained_gguf("model_minority_report", tokenizer, quantization_method = "q4_k_m")
```

## Real-World Application: UN HIPS Monitoring

### Scenario
The **UN Hazard Information Profiles (HIPS)** require precise tracking of hazards (e.g., *Peste des petits ruminants*) across multilingual situation reports. A standard LLM might hallucinate terms or use colloquial synonyms, breaking data interoperability.

### Solution with Fine-Tuned Model
By training a model on the Minority Report dataset, you enforce the **Controlled Vocabulary**.

**Example Workflow:**

1.  **Input (Situation Report)**:
    > "Une épidémie sévère touchant les chèvres a été signalée au Niger."

2.  **Prompt**:
    > "Identify the standard hazard term from the text in French. Output the HIPS code if known."

3.  **Model Output (Fine-Tuned)**:
    > "Hazard: **Peste des petits ruminants** (HIPS: BI0310)"

**Why this matters**:
*   **Standardization**: The model "knows" that "épidémie touchant les chèvres" maps to the formal term *Peste des petits ruminants*, not generic "Goat Plague".
*   **Multilingual Interoperability**: It can map a Russian report ("ППР") to the same English concept ID seamlessly because it was trained on the parallel corpus.

## Definición
Malware es un término sumario que engloba las diversas formas de software malicioso diseñado para infiltrarse e infectar computadoras, típicamente sin el conocimiento del propietario (ITU, 2008).

## Referencias principales
ITU, 2008. *Estudio de la UIT sobre los aspectos financieros de la seguridad de redes: Malware y Spam*. Unión Internacional de Telecomunicaciones (UIT). Accedido el 31 de octubre de 2024.

## Anotaciones
### Descripción científica adicional
Malware es una palabra compuesta de “malicious software”. Se refiere a software diseñado para causar daños, interrumpir operaciones o obtener acceso no autorizado a sistemas, redes o dispositivos informáticos. Incluye varios tipos de software dañino, cada uno con características y métodos de propagación específicos.

Malware suele categorizarse en **familias** (tipo particular de malware con características únicas) y **variantes** (versión distinta de código dentro de una familia) (ITU, 2008).

Tipos de malware:
- **Virus**: Se adjuntan a programas legítimos y se propagan de dispositivo a dispositivo cuando se ejecutan esos programas.
- **Gusanos**: Se replican para propagarse a otros sistemas sin requerir un programa huésped.
- **Caballos de Troya**: Se hacen pasar por software benigno para engañar a los usuarios y comprometer el sistema.
- **Ransomware**: Los ciberdelincuentes cifran datos y exigen pago a cambio de las llaves de descifrado.
- **Spyware**: Recopila información en secreto al monitorear las actividades de los usuarios.
- **Wiper** (eliminador): Elimina o corrompe datos permanentemente en máquinas infectadas.
- **Rootkits**: Permiten acceso no autorizado ocultando su presencia dentro del sistema.

La historia de malware se remonta al menos a los años 80, cuando surgieron los primeros ejemplos (Milošević, 2014). Incidentes notables incluyen el gusano Morris (1988) y el ataque ransomware WannaCry (2017).

## Vectores de entrega
Malware puede propagarse a través de múltiples vectores: adjuntos de correo electrónico (especialmente en ataques de phishing), sitios web maliciosos, descargas de software infectado y vulnerabilidades de red. Los atacantes emplean inteligencia artificial para crear malware avanzado, como versiones polimórficas que cambian su código para evadir la detección.

### Malware sin archivos
Esta amenaza creciente evaden los antivirus tradicionales al aprovechar herramientas legítimas del sistema y archivos existentes. Se propaga mediante phishing, sitios comprometidos o ataques de “watering hole”, y puede robar información sensible y propagarse por la red, generando riesgos de escala nacional e internacional cuando otros sistemas son objetivo deliberado o accidental.

## Impacto histórico
Un caso emblemático es el gusano **Stuxnet** (2010), diseñado para sabotear instalaciones nucleares en Irán. Este sofisticado malware demostró el potencial de las armas cibernéticas para causar daños físicos y perturbar la seguridad nacional (Zetter, 2014).

## Métricas y límites numéricos
Se detectaron más de **1 000 000 000** variantes de malware en 2023 (Statista, 2024).

## Instrumentos internacionales
- Convención de Budapest sobre el cibercrimen del Consejo de Europa: directrices para la cooperación internacional en la lucha contra el cibercrimen, incluyendo delitos de creación, distribución o uso de malware.
- Código de Práctica del Proceso Pall Mall para los Estados (actualizado 2025): diálogo global para abordar la proliferación y el uso irresponsable de herramientas y servicios de intrusión cibernética.

## Impulsadores
- Vulnerabilidades de software  
- Aumento de la dependencia digital  
- Economía del cibercrimen  
- Tensiones geopolíticas  

## Impactos
Los ataques de malware pueden escalar a amenazas de escala nacional al apuntar a infraestructuras críticas o sistemas gubernamentales, como se evidenció con Stuxnet.

## Contexto multi‑riesgo
Malware puede causar interrupciones de servicios, pérdidas financieras, pérdida de ingresos, pérdida o robo de datos y/o inteligencia, daño a la infraestructura de TI, daños reputacionales y pagos de rescates y multas regulatorias. También puede aumentar los costos de recuperación y la pérdida de confianza pública. Los sistemas de software están interconectados con sistemas físicos; malware puede usarse para acceder a sistemas que abran otros riesgos, como la manipulación de químicos en sistemas de distribución de agua. Ejemplos: Florida (2021), California (2021), Israel (2020) y EE. UU. (2016) (Sikder et al., 2023).

## Gestión de riesgos
Defenderse contra malware requiere:  
1. Concienciación del usuario (la mayoría del malware se activa tras una interacción del usuario).  
2. Medidas técnicas: antivirus y anti‑malware, actualizaciones regulares del sistema, y segmentación de red.  
Estas medidas proporcionan capas adicionales de protección, pero no sustituyen la concienciación del usuario.

## Monitoreo
El monitoreo de amenazas se lleva a cabo a nivel internacional, nacional, sectorial y organizacional, compartiendo información sobre identificación de malware mediante diversos mecanismos para mitigar los riesgos.

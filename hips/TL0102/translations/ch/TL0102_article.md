### Definition

数据泄露是指当企业或组织负责的数据因安全事件导致机密性、可用性或完整性受到破坏时的情况（欧洲委员会，无日期）。

### Annotations

#### Additional scientific description

在网络安全领域，数据泄露指的是在未获得授权的情况下访问、披露或检索敏感、受保护或机密数据的事件。此类事件可能涉及个人数据、财务信息、知识产权或其他关键数据，进而对个人或组织造成潜在危害。数据泄露可表现为多种形式，例如：  
- 对数据库的未授权访问  
- 在传输过程中的数据拦截  
- 内部威胁  

个人数据泄露危及可识别个人信息（PII），如姓名、地址、社会保险号码和财务细节，导致身份盗窃和隐私侵犯。非个人数据泄露涉及商业专有信息、知识产权或其他敏感组织数据的曝光，可能导致财务损失和竞争劣势。  

当数据泄露针对关键基础设施、政府机构或大型数据仓库时，可升级为国家级危害。例如 2015 年美国人事管理局的数据泄露事件，超过 2100 万联邦雇员的敏感信息被泄露，给国家安全和情报行动带来重大风险（Koerner，2016）。2022 年共发生 1,774 起数据泄露事件，影响全球 3.92 亿人，暴露受法律保护的身份信息，识别了在金融、医疗保健和社会服务等部门使用服务的人群。这些数据泄露还具有显著的经济影响，其成本每年增长 20% 以上，现占全球 GDP 的 4-6%（WEF，2023）。人因是最常见的脆弱性，占 68% 与无意人类行为相关的泄露。社会工程策略经常欺骗个人点击恶意链接或共享敏感信息，随后被利用（Verizon，2024）。

### Key relevant UN convention / multilateral treaty

国际法律工具包括欧洲委员会的《网络犯罪公约》（布达佩斯公约），促进打击网络犯罪的国际合作，包括未授权访问和数据干扰。 另外，许多国家和地区法规处理个人数据泄露，例如欧盟的《通用数据保护条例》、加利福尼亚消费者隐私法、加拿大个人信息保护与电子文件法等。

### Drivers

- 缺乏网络安全投资  
- 人为失误  
- 第三方风险  
- 通过 API 增加攻击面  

未来的泄露很可能聚焦于处理大量客户和敏感数据的主要科技公司。此类目标增加源于 API 的广泛使用、数据数字化的增长以及零日漏洞未被发现。

### Impacts

2023 年全球数据泄露景观明显加剧，较 2022 年高峰期数据泄露数量增长 72%（WEF，2024）。尽管泄露事件增多，泄露通知变得更不清晰。2022 年美国 66% 的公共通知未提供受影响受害者信息或攻击细节，主要由于缺乏国家隐私法。 此外，泄露后服务（如信用监控）往往未与减少未来曝光风险的建议步骤保持一致（WEF，2023）。

### Multi‑hazard context

数据泄露的后果可能严重，导致财务损失、声誉损害、法律与监管影响、运营中断以及个人伤害（身份盗窃、隐私侵犯等）或组织伤害。数据泄露可进一步使组织和个人面临网络攻击（恶意软件、数据操纵）及供应链中断（第三方供应商与合作伙伴安全问题级联）等风险。

### Risk Management

为有效解决和减轻这些风险，组织应采取主动措施保护消费者信息并定义清晰、可操作的关键绩效指标。鼓励公司实施多因素认证服务，并合作制定行业标准。保险公司和监管机构可通过提供激励措施促进数据保护与企业安全。网络保险可通过提供更低保费、免费预泄露服务以及持续承保适应不断变化的威胁和演变的网络风险管理实践来支持该方法。  

**最合适的减缓数据泄露风险策略取决于攻击实施方式**：

#### 凭证盗窃攻击  
- 实施强密码策略与多因素认证（MFA）  
- 使用密码管理器并定期更新凭证（Kost，2024）

#### 网络钓鱼攻击  
- 定期开展员工培训与意识计划  
- 实施电子邮件安全措施（防止冒充与加密）

#### 漏洞利用  
- 维护稳健的补丁管理系统  
- 定期进行漏洞评估与渗透测试（Pen‑Test）  
- 实施网络分段以隔离关键系统
